Almacenaje, memoria y archivos



Introducción

Esta actividad se abordará todo lo relacionado con el almacenaje, la memoria y los archivos. Siempre se cree que almacenamiento y memoria son términos que significan lo mismo, sin embargo, esto es una creencia errónea.
Memoria se refiere al componente que se encuentra dentro de la computadora, el cual permite acceder a los datos almacenados en el dispositivo a corto plazo. Asimismo, la velocidad y el rendimiento del sistema dependen de la cantidad de memoria instalada en el computador.
El almacenamiento es el componente de la computadora que permite almacenar y acceder a los datos a largo plazo. Se da en forma de unidad en estado sólido o en disco duro. Permite acceder y almacenar aplicaciones, sistema operativo y archivos por un tiempo indefinido.

¿Qué tipos de memoria y sistemas de archivos manejan y cómo operan?



Tipos de memorias

  • Linux, tiene dos espacios de memoria: el espacio de memoria del usuario y el espacio de memoria del kernel.
  • El sistema operativo DOS cuenta con memoria convencional, memoria extendida, memoria paginada y memoria alta.
  • Los ordenadores con sistema operativo Windows, Ubuntu o Linux disponen de esta clase de memoria, que es virtual. Swap se puede traducir por “espacio de intercambio”. Es bastante parecida a la memoria caché, pero a ella recurre exclusivamente el sistema operativo, y no el resto de los componentes del ordenador.
  • La gestión de memoria en Windows es de memoria virtual con paginación, es decir, se encarga del intercambio de información entre la memoria y el Disco Duro.

Sistema de archivos

    Un sistema de archivos es el sistema de almacenamiento de un dispositivo de memoria, que estructura y organiza la escritura, búsqueda, lectura, almacenamiento, edición y eliminación de archivos de una manera concreta.
    Windows, utiliza el sistema de archivos NTFS.
    NFTS es un sistema de archivos de Windows NT incluido en las versiones desde Windows 2000 hasta Windows 10. Está basado en el sistema de archivos HPFS de IBM/Microsoft usado en el sistema operativo OS/2, y también tiene ciertas influencias del formato de archivos HFS diseñado por Apple.

    Linux ofrece distintos sistemas de archivos como: Ext, Ext2, Ext3, Ext4, swap, ReiserFS, XFS, JFS, UFS, ISO9660, FAT, FAT32 o NTFS.
  • ext2: Hasta hace poco era el sistema estándar de Linux. Tiene una fragmentación muy baja, aunque es algo lento manejando archivos de gran tamaño. Fue la continuación del sistema de ficheros ext, implementado en 1992 e integrado en Linux 0.96.
  • ext3: Es la versión mejorada de ext2, con previsión de pérdida de datos por fallos del disco o apagones.
  • ext4: Es la última versión de la familia de sistemas de ficheros ext. Sus principales ventajas radican en su eficiencia y en la ampliación de los límites de tamaño de los ficheros, ahora de hasta 16TB.
  • ReiserFS: Es el sistema de ficheros de última generación para Linux. Organiza los ficheros de tal modo que se agilizan mucho las operaciones con estos. El problema de ser tan actual es que muchas herramientas lo soportan.
  • swap: Es el sistema de ficheros para la partición de intercambio de Linux. Todos los sistemas Linux necesitan una partición de este tipo para cargar los programas y no saturar la memoria RAM cuando se excede su capacidad.

    • Mac cuenta con diversos sistemas de archivos, dentro de los cuales están los siguientes:
  • Apple File System (APFS): el sistema de archivos usado en macOS 10.13 o posterior. Es el sistema de archivos predeterminado para computadoras Mac con macOS 10.13 o posterior, posee un encriptado sólido, reparto de espacio, instantáneas, cambio de tamaño de directorio rápido y fundamentos de sistema de archivos mejorados.
  • MS-DOS (FAT) y ExFAT: sistemas de archivo compatibles con Windows.
  • MS-DOS (FAT): se utiliza para volúmenes de Windows de hasta 32 GB.
  • ExFAT: se utiliza para volúmenes de Windows de más de 32 GB.


Sistemas Operativos de red

NFS (Network File System - Sistema de archivos de red).- es un mecanismo para almacenar archivos en una red. Es un sistema de archivos distribuido que permite a los usuarios acceder a los archivos y directorios ubicados en sistemas remotos y tratar dichos archivos y directorios como si fueran locales.
Un sistema operativo de red es una aplicación de software que funciona a modo de plataforma para mejorar tanto el funcionamiento de un ordenador como el de varios ordenadores dentro de una misma red interconectada. De este modo, un sistema operativo de red es capaz de posibilitar la comunicación entre ordenadores pertenecientes a la misma red de equipos.

LANtastic de Artisoft es un sistema operativo de red punto a punto para DOS, Windows y OS/2. Su principal particularidad reside precisamente en sus redes punto a punto, lo cual quiere decir que la comunicación únicamente entre dos equipos.

LAN Manager de Microsoft, es un sistema operativo desarrollado por Microsoft e IBM, ejecutado bajo OS/2 que conecta equipos con DOS, OS-2 y UNIX. Además de agrupar los servidores de archivos de dominios, es apta para equipos de macrocomputadoras y cuenta con la particularidad de ofrecerse tanto para servidor dedicado como no dedicado.

Windows Server, es un sistema operativo lanzado por Microsoft y está desarrollado en C, C++ y asembler. Es un sistema multiproceso y que admite varios usuarios. Sin embargo, es más utilizado por empresas ya que el costo de la licencia es un tanto elevado.


Importancia de la administración de memoria

La memoria es uno de los elementos más importantes para que todo funcione correctamente, sin ella nuestros dispositivos electrónicos tal vez ni siquiera podrían arrancar; tal es la importancia que este componente eléctrico tiene en la estructura de los aparatos electrónicos.
La gestión de la memoria principal de una computadora es una tarea de suma importancia para el funcionamiento de esta. La calidad de la gestión de la memoria es crucial para las prestaciones del sistema. El recolector de basura es la asignación y liberación automática de los recursos de memoria para un programa.


Control de los espacios disponibles y ocupados en la memoria

Su funcionamiento consiste en:
Inicialmente toda la memoria(a excepción de la partición del S.O. se encuentra disponible para los procesos, similando un gran hueco.
Cuando llega un proceso se guarda en un hueco que esté libre y el espacio que se dejó libre representa un hueco más.
Cada una de las zonas de la memoria que se encuentren ocupadas representan una partición.
Una vez que el proceso ya ha terminado libera su zona de la memmoria y genera un hueco, mismo que se fusiona con los que lo rodean.
Finalmente se conserva una tabla donde se almacenan todas las partes de memoria utilizadas y también las libres, así como la cola de los procesos que están esperando un hueco.


Problemas frecuentes en la administración de memoria

Como la memoria disponible estará compartida por varios procesos, el programador no puede conocer mientras desarrolla qué área de memoria se asignará al proceso que está programando. Cuando se ejecute el programa, el SO asignará un bloque de memoria libre, que podrá ser diferente en cada ejecución.

No resulta conveniente que un proceso pueda acceder al espacio de memoria asignado a otro proceso.

En ocasiones algunos procesos necesitan intercambiar datos. Estos procesos pueden querer compartir un área de memoria a la que ambos tengan acceso.

Normalmente en un sistema informático la memoria principal está organizada de forma lineal como una secuencia de posiciones de memoria. Del mismo modo la memoria secundaria se puede ver como una secuencia de bloques. El sistema gestor de memoria debe permitir organizar lógicamente partes de la memoria para acercarse a la visión del programador.

Físicamente la memoria está organizada en dos espacios claramente diferenciados:
  • Memoria principal rápida, volátil y escasa
  • Memoria secundaria lenta, persistente y abundante

Además de ser el encargado de administrar y facilitar el uso de las memorias periféricas, ya sean principales, secundarias o terciarias. También se abordó el tipo de sistema de archivos con el que cuentan algunos sistemas operativos de PC y móviles.
